API Documentation

Getting Started

To get started with the API, follow these steps:

  1. Clone the repository to your computer.
  2. Go to the project directory in your terminal.
  3. Run the following command to install the dependencies:
    4. npm install
  4. Start the application using the following command when it has finished installing:
    5. npm start
  5. Once the server is running, open your browser and go to http://localhost:3000/ to access the website.

1. Events API

GET /api/events

Get a list of event IDs and titles.

Query Parameters:

  • title (optional) - Filter events by title.
  • location (optional) - Filter events by location.
GET /api/events?title=Concert&location=Venue

Example Response:

[
{
  "id": "1",
  "title": "Concert A"
},
{
  "id": "2",
  "title": "Concert B"
}
]

Error Response:

404 Not Found - If no events match the filters.

GET /api/events/:id

Get event details by event ID.

GET /api/events/{id}

Example Response:

{
  "id": "1",
  "title": "Concert A",
  "location": "Venue A",
  "time": "2025-01-30T20:00:00Z",
  "capacity": 500,
  "description": "A fun concert",
  "imageURL": "https://example.com/concert.jpg"
}

Error Response:

404 Not Found - If the event with the given ID does not exist.

POST /api/events

Add a new event.

Request Body:

{
  "title": "Concert C",
  "location": "Venue B",
  "time": "2025-02-10T18:00:00Z",
  "capacity": 300,
  "description": "A new concert",
  "imageURL": "https://example.com/concert-c.jpg"
}

Example Response:

{
  "message": "Event created successfully",
  "event": {
    "id": "3",
    "title": "Concert C",
    "location": "Venue B",
    "time": "2025-02-10T18:00:00Z",
    "capacity": 300,
    "description": "A new concert",
    "imageURL": "https://example.com/concert-c.jpg"
  }
}

Error Response:

400 Bad Request - If the required fields (title, location, time) are not provided.

2. Musicians API

GET /api/musicians

Get a list of musicians' IDs and names.

Query Parameters:

  • name (optional) - Filter musicians by name.
  • location (optional) - Filter musicians by location.
GET /api/musicians?name=Musician&location=New%20York

Example Response:

[
{
  "id": "1",
  "name": "Musician A"
},
{
  "id": "2",
  "name": "Musician B"
}
]

GET /api/musicians/:id

Get musician details by musician ID.

GET /api/musicians/{id}

Example Response:

{
  "id": "1",
  "name": "Musician A",
  "instruments": ["Guitar", "Piano"],
  "about": "A talented musician",
  "location": "Newcastle",
  "image": "https://example.com/musician-a.jpg"
}

Error Response:

404 Not Found - If the musician with the given ID does not exist.

POST /api/musicians

Add a new musician.

Request Body:

{
  "name": "Musician C",
  "instruments": ["Drums"],
  "about": "A drummer",
  "location": "Durham",
  "image": "https://example.com/musician-c.jpg"
}

Example Response:

{
  "message": "Musician added successfully",
  "musician": {
    "id": "3",
    "name": "Musician C",
    "instruments": ["Drums"],
    "about": "A drummer",
    "location": "Durham",
    "image": "https://example.com/musician-c.jpg"
  }
}

Error Response:

400 Bad Request - If the required fields (name, instruments) are not provided.

3. User Profile API

GET /api/user

Get the user profile details.

GET /api/user

Example Response:

{
  "name": "User A",
  "instruments": ["Guitar"],
  "about": "Music enthusiast",
  "location": "Chicago",
  "image": "https://example.com/user-a.jpg"
}

POST /api/user

Update the user profile.

Request Body:

{
  "name": "User B",
  "instruments": ["Piano"],
  "about": "Music lover",
  "location": "Durham",
  "image": "https://example.com/user-b.jpg"
}

Example Response:

{
  "message": "User profile updated successfully",
  "user": {
    "name": "User B",
    "instruments": ["Piano"],
    "about": "Music lover",
    "location": "Durham",
    "image": "https://example.com/user-b.jpg"
  }
}

Error Response:

400 Bad Request - If the required field (name) is not provided.
